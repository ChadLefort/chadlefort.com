## Problem

Three frontend apps shared no UI primitives. Two apps were built on Chakra UI,
one with heavy use of custom CSS modules layered on top, and the third on PrimeReact.
Accessibility was inconsistent (focus management, ARIA wiring, keyboard
support varied component by component), visual language heavily drifted between
apps, and every new feature reinvented similar components.

## Solution

Architected an internal component library built on React Aria Components and
Tailwind Variants + UnoCSS for styling, then methodically replaced existing
component libraries and the bespoke components across all three apps.
Shipped 40+ components covering the full surface area. Every component ships with
Storybook coverage and tests. Some components have multiple variants allowing for
flexibility. Organized in Storybook by category:

### **Actions**

- Button
- Link Button

### **Data Display**

- Badge
- Date Span
- Table (Base)
- Table (Grouped)
- Tag

### **Layout**

## Impact

- Replaced three disparate UI sources with one cohesive system.
- All legacy UI libraries were fully removed from the codebase.
- Raised the a11y bar across the team.
- Visual consistency across all three apps.
- Faster feature work with engineers composing from a known library.
