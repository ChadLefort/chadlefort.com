## Problem

Three frontend apps shared no UI primitives. Two apps ran on Chakra UI, one with heavy custom CSS modules layered on top, and the third on PrimeReact. Accessibility varied (focus management, ARIA wiring, and keyboard support differed component by component), the visual language drifted between apps, and every new feature reinvented similar components.

## Solution

I architected an internal component library built on React Aria Components and Tailwind Variants + UnoCSS for styling, then replaced existing component libraries and bespoke components across all three apps. I shipped 40+ components covering the full surface area. Each component ships with Storybook coverage and tests. Some components ship with multiple variants for flexibility. ## Impact

- Replaced three disparate UI sources with one cohesive system.
- Removed all legacy UI libraries from the codebase.
- Raised the a11y bar across the team.
- Brought visual consistency across all three apps.
- Feature work moves faster with engineers composing from a known library.
