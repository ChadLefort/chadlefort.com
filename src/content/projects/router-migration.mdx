## Problem

Three frontend apps ran on a fragmented routing stack: one on Router 5, another on an older React Router, and the team reinvented shared navigation patterns per app. No types on routes. The team validated search params by hand. No shared loader or cache story existed with our data layer. MobX stores observed the legacy routers and locked up much of the app state. Any router rewrite would cascade through hundreds of observer wrapped components.

## Solution

I sequenced the work in two phases shipped incrementally behind a migration shim, with zero customer downtime:

#### 1. **State migration (MobX → Zustand).**

I consolidated MobX stores under a single root, then migrated each store to Zustand backed shared state. I moved server state to TanStack Query v5. That removed observer wrappers and made stores more testable.

#### 2. **Router migration (Router 5 / React Router → TanStack Router).**

I introduced TanStack Router alongside the legacy router with a migration hook, Link, and helpers so the team could move routes over one at a time. Once the team ported the last route, I ripped out Router 5 and finalized typed routes, Zod validated search params, and route level loaders that integrate with TanStack Query.

## Impact

- Unified three apps on one type safe routing stack with shared patterns for links, search params, and loaders.
- End to end type safety across routes and search params eliminated a class of runtime navigation bugs.
- Killed MobX entirely, simplified store code, and made stores unit testable without React render trees.
- I shipped incrementally so the app stayed shippable on every commit. Users saw no freeze or downtime window.
